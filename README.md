## 🗂️ Convention

### 🌿 Git Branch

| 브랜치                      | 용도          |
| --------------------------- | ------------- |
| `main`                      | 배포용 브랜치 |
| `dev`                       | 개발용 브랜치 |
| `feat/#issue-number/기능명` | 기능 개발     |

<details>
<summary> 🚀 개발 프로세스 </summary>
  
1. **이슈 기반 브랜치 생성**
   * 모든 작업 시작 전 GitHub Issue를 생성합니다. 
   * 해당 이슈 번호를 기준으로 dev 브랜치에서 기능 별 이슈 브랜치로 분기합니다. (<code style="color: #e5534b">**feat/기능명/#issue-number**</code>)

2. **작업 및 PR 생성**
   - 기능 구현이 완료되면 PR을 생성합니다.
   - 팀원 중 2명 이상의 Approve이 있을 때, develop 브랜치로 병합합니다.
   </details>

<br />

### 🔖 Commit Convention

| 태그       | 내용                   |
| ---------- | ---------------------- |
| `init`     | 초기 세팅              |
| `feat`     | 새로운 기능 추가       |
| `fix`      | 버그 수정              |
| `docs`     | 문서 수정              |
| `style`    | 스타일 코드 변경       |
| `type`     | 타입 수정              |
| `refactor` | 리팩토링               |
| `chore`    | 기타 작업              |
| `test`     | 테스트 코드            |
| `rename`   | 파일 또는 폴더 명 수정 |
| `remove`   | 파일 또는 폴더 삭제    |
| `HOTFIX`   | 긴급 수정              |

ex) `init: 커밋 메시지`

<details>
<summary> 🔀 PR Convention </summary>
- 하나의 PR에는 하나의 기능만 담는 것을 원칙으로 합니다.
- PR 제목은 `[Type] 작업 내용` 형식으로 작성합니다.
- PR과 Issue에는 라벨을 꼭 설정합니다.
- 최소 2명 이상의 approve 이후 merge합니다.
</details>

<br />

### 💻 Code Convention

<details>
<summary>🌐 컴포넌트</summary>

- 폴더와 파일은 `camelCase`, 컴포넌트는 `PascalCase`를 사용합니다.
- 컴포넌트 `*.tsx`는 `default export`를 사용합니다.
- `index.ts` barrel export와 `named + default` 혼용 export는 금지합니다.
- 의미없는 div 태그 사용은 지양합니다.

</details>

<details>
<summary>💡 변수/상수</summary>

- 상수는 `UPPER_SNAKE_CASE`를 사용합니다.
- boolean 값은 `is`, 복수 데이터는 `s` 네이밍을 사용합니다.
- 줄임말보다는 의미가 분명한 이름을 사용합니다.

</details>

<details>
<summary>🔑 함수 & 훅</summary>

- 함수는 화살표 함수를 사용합니다.
- 이벤트 핸들러에만 `handle` 접두어를 붙입니다.
- 훅·유틸·상수는 `named export`를 사용합니다.
- 훅은 `use-` 접두어를 사용합니다.
- import는 가장 짧은 alias를 사용하고, `@/` prefix와 2단계 이상 상대경로는 지양합니다.

</details>

<details>
<summary>🧩 타입</summary>

- 타입은 `PascalCase`를 사용합니다.
- 가능하면 `type`보다 `interface`를 우선 사용합니다.
- Props 타입은 `~Props`, 일반 타입은 `~Types` 네이밍을 사용합니다.
- any 타입을 금지합니다.

</details>

<details>
<summary>🎨 스타일</summary>

- `reset.css`를 적용하고 기본 단위는 `rem`을 사용합니다.
- 컴포넌트 스타일은 같은 위치의 `*.css.ts` 파일에 작성합니다.
- 스타일 import는 `import * as styles from './Component.css';` 형식을 사용합니다.
- wrapper가 필요할 때는 `container` 네이밍을 우선 사용합니다.
- asset 파일명은 피그마 추출명을 유지하고, import 변수명은 `img + camelCase`를 사용합니다.

</details>
